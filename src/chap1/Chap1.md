### module
-   크기에 상관없이 클래스나 패키지 , 라이브러리와 같이 프로그램을 구성하는 임의의 요소를 의미
-   목적
    -   실행중에 제대로 동작
    -   변경에 유연
    -   코드를 읽는 사람과의 의사 소통
    
- chap1에 코드들은 의존관계가 너무 강해 하나 변경시 이를 사용하는 곳에 변경이 일어난다. (즉 객체간의 결합도가 높다)  

-   캡슐화
    -   개념적이나 물리적으로 객체 내부의 세부적인 사항을 감추는 것을 캡슐화라고 한다.
    -   객체의 자율성을 높인다.
        -   자율성이란 
            -   자신의 상태를 스스로 관리 하는것
   
- 객체 내부의 상태를 캡슐화 하고 객체 간에 오직 메시지를 통해서만 상호 작용하도록 해야한다.

-응집도
    -   밀접하게 연관된 작업만을 수행하고 연관성 없는 작업은 다른 객체에게 위임하는 객체를 가리켜 응집도가 높다고한다.
   
- 객체의 응집도를 높이기 위해서는 객체 스스로 자신의 데이터를 책임져야한다.
    -   그말은 즉 자신의 데이터만으로 처리하고 할 수 없는 것은 위임하라는 것이다. 만약 그렇지 않고
         다른 객체의 데이터를 직접 접근하여 사용한다면 응집도는 낮아지고 결합도가 증가하기에 변경이 생기면 사이드 이펙트가 생긴다.

-   절차 지향
    -   프로세스와 데이터를 별도의 모듈로 나누고 데이터를 중심인 것이 절차 지향 방식

-   객체 지향
    -   객체 기준
       -    데이터와 프로세스를 가지고 있는 객체를 기준으로
    -   책임은 객체가 가진 기능이다.
    -   핵심은 적절한 객체에 적절한 책임을 할당하는 것
